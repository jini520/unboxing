# UI 디자인 가이드

> 대상: Expo (React Native). 웹 Tailwind가 아니라 RN `StyleSheet`/스타일 객체 기준.
> **테마는 시스템 외형(light/dark)을 따르며 라이트가 설계 기준**(→ ADR-016). 색은 하드코딩하지 말고 아래 **시맨틱 토큰**(라이트/다크 값)으로 쓴다.
> 색상·수치는 **초안** — Phase 1 UI 구현 step에서 확정한다. 원칙·안티패턴·접근성·상태/테마 규칙은 확정.
> 기능/플로우/스토어 정책은 `PRD.md`, 데이터/엔드포인트는 `ARCHITECTURE.md`.

## 디자인 원칙
1. **도구처럼 보여야 한다.** 마케팅이 아니라 매일 쓰는 배송 추적 대시보드. 신뢰감 있고 깔끔하게.
2. **시스템 테마를 따르고, 라이트를 기준으로 설계한다.** 대중 택배 앱의 표준 기대치는 라이트. 다크는 동등 품질로 지원(다크 전용/디폴트 아님).
3. **마찰 최소.** 등록은 두세 탭 안에. 불필요한 모달·온보딩 단계 금지.
4. **상태는 색에만 의존하지 않는다.** 색 + 아이콘 + 텍스트 라벨을 항상 함께(접근성).

## AI 슬롭 안티패턴 — 하지 마라
(RN 맥락으로 적응. CSS 용어는 RN 대응으로 읽을 것.)

| 금지 사항 | 이유 |
|-----------|------|
| glass morphism (expo-blur로 카드 배경 흐림 장식) | AI 템플릿의 가장 흔한 징후 |
| 그라데이션 텍스트 | AI가 만든 SaaS 랜딩의 1번 특징 |
| "Powered by AI" 류 배지 | 기능이 아니라 장식. 사용자에게 가치 없음 |
| 글로우 애니메이션 (네온 shadow 펄스) | 네온 글로우 = AI 슬롭 |
| 보라/인디고 브랜드 색상 | "AI = 보라색" 클리셰 |
| 모든 카드에 동일한 큰 borderRadius(2xl 류) | 균일한 둥근 모서리는 템플릿 느낌 |
| 과한 그림자/elevation 남발 | 라이트에서 특히 지저분. 그림자는 아주 옅게만 |
| 배경 그라데이션 orb (blur 원형 장식) | 모든 AI 랜딩 페이지에 있는 장식 |

## 화면 구성

| 화면 | 내용 |
|---|---|
| **목록**(주화면) | 송장 카드 스택 + 당겨서 새로고침 + (클립보드 제안 배너) + 등록 진입 + 마지막 업데이트 표기 |
| **상세** | 단계 배지 + 타임라인(실시간 조회) + 택배사/번호 + 삭제 |
| **등록** | 번호 입력 → 택배사 자동추정/확인(드롭다운) → 등록. 미지원이면 딥링크 카드 |
| **온보딩** | 푸시 권한 사전 안내(priming) → 시스템 요청 |
| **설정** | 알림 토글 · 테마(시스템/라이트/다크) · 개인정보처리방침 링크 · **모든 데이터 삭제** · 앱 버전 |
| **빈 상태** | 가치 제안 한 줄 + 첫 등록 CTA (중앙 정렬 허용) |

## 단계 배지 (색 + 아이콘 + 라벨 — 색 단독 금지)

| 단계 | 아이콘(예) | 라벨 |
|---|---|---|
| `배송완료` | ✓ check | 배송완료 |
| `배송출발` | truck | 배송출발 |
| `예외` | alert | 예외 |
| `등록`·`집화`·`이동중`·`기타` | dot/box | 각 단계명 |
| `미등록` | clock | 미등록 |

색은 위 "단계(시맨틱) 색"을 단일 출처로(ARCHITECTURE 상태 정규화 7단계와 1:1).

## 상태별 UI (로딩/빈/에러 — 누락 금지)

| 상태 | 표시 |
|---|---|
| 로딩(목록) | 스켈레톤 카드(좌측 정렬), 캐시 있으면 캐시 먼저 + 상단 새로고침 인디케이터 |
| 로딩(상세 타임라인) | 단계 배지는 즉시(캐시), 타임라인은 스켈레톤 |
| 빈 목록 | 간결한 가치 제안 + 등록 CTA(중앙 정렬) |
| 네트워크 오프라인 | 상단 **오프라인 배너**(비파괴적), 캐시 표시, 변경 액션 비활성+사유 |
| 조회 실패(상세 upstream) | 마지막 알려진 단계 + "타임라인을 못 불러왔어요, 다시 시도" |
| 미지원 택배사 | "자동 추적 미지원" + 택배사 조회 **딥링크** 버튼 |
| 등록 실패 | 인라인 에러(코드 노출 금지) + 재시도, 입력값 보존 |
| 알림 비활성(권한 거부) | 상단 작은 배너 "알림 꺼짐 — 켜기"(설정 유도) |

## 송장 카드 해부
```
[단계 배지]  택배사명 · 번호 끝 4자리            [D-day · 방금 업데이트]
한 줄 요약(현재 위치/단계 문구)
```
- 좌측 정렬. 단계 배지=색+아이콘+짧은 라벨.
- 스와이프 → 삭제(**확인 다이얼로그 + Undo 토스트**). 탭 → 상세.
- 택배사 식별: 택배사명 + (선택) 택배사 색 점/로고 — 로고 사용 시 **상표 가이드 준수**(임의 변형·왜곡 금지, 미확보 시 텍스트/중립 아이콘으로 대체).

## 타임라인 컴포넌트
- 세로 단계 점(현재 단계 강조) + 시각(KST, 상대+절대) + 설명/허브명(위치는 비개인정보).
- 최신이 위. 단계 전환점 라벨 강조.

## 알림 UI
- 단계별 카피는 `PRD.md` 알림 정책 참조. 본문 ≤ payload 4096B.
- **그룹화**: 여러 송장 알림은 묶음/요약(과알림 방지, 조용시간과 결합).
- Android: "배송 상태" notification channel 분리.
- 알림 탭 → 해당 **상세 딥링크**(payload `shipment_id`).

## 설정 / About 화면
- 항목: 알림 켜기/끄기, 테마(시스템·라이트·다크), 개인정보처리방침(외부 링크), **모든 데이터 삭제**(빨강·확인 다이얼로그·복구 불가 명시 → `DELETE /me`), 앱 버전.
- "모든 데이터 삭제"는 파괴적 액션 스타일(`예외` 색), 실행 후 결과 토스트.

## 컴포넌트 (초안 — 토큰 참조)
### 카드 (송장 항목)
```
borderRadius: 8, backgroundColor: bg/surface, borderWidth: 1, borderColor: border, padding: 16
(라이트: 옅은 그림자 1단계만 / 다크: 그림자 없음)
```
### 버튼
```
Primary: backgroundColor text/primary(반전 surface 위), 라벨 대비색, borderRadius 8
Text:    color text/secondary (press → text/body)
Destructive: color = 단계 '예외' 색
```
### 입력 필드 (운송장 번호)
```
backgroundColor bg/secondary, borderWidth 1, borderColor border, borderRadius 8, paddingHorizontal 16, paddingVertical 12
```

## 레이아웃
- 정렬: 좌측 정렬 기본. 중앙 정렬은 빈 상태에서만.
- 간격: 항목 간 8~12, 섹션 간 24~32.
- 리스트가 주 화면 — 카드 스택 + 당겨서 새로고침. **정렬**: 진행 중 우선, 임박·예외 강조.

## 타이포그래피 (초안)
| 용도 | 스타일 |
|------|--------|
| 화면 제목 | 28~32, weight 600, `text/primary` |
| 카드 제목(택배사·번호) | 14, weight 500, `text/secondary` |
| 본문/타임라인 | 14, `text/body` |

## 마이크로카피 / 톤
- 친근한 한국어. **에러 코드/기술 용어 노출 금지**(내부 code는 로그만).
- 상대 시간("방금 업데이트", "2시간 전") + 절대 시각 보조.
- 실패 카피 예: "잠시 후 다시 시도해 주세요". 빈 상태: "운송장만 넣어두면 상태가 바뀔 때 알려드려요".

## 인터랙션
- **당겨서 새로고침**: 목록·상세. 오프라인이면 안내.
- **스와이프 삭제 + Undo**: 확인 다이얼로그 후 토스트 Undo(실수 복구), 서버 반영은 Undo 만료 후.
- **클립보드 제안**: 등록 화면 진입 등 명시 시점에만, 제안만(자동 등록 금지 — iOS 배너 정책, PRD 권한).
- **토스트/에러**: 일시 오류는 비차단 토스트, 영구 오류는 인라인.

## 접근성 (확정)
- **색 단독 금지**: 상태=색+아이콘+텍스트.
- **대비**: 라이트·다크 **양쪽 모두** 본문/배경 WCAG AA 이상(위 토큰은 이를 만족하도록 선정).
- **다이내믹 타입**: 시스템 글자 크기 존중(고정 px 남용 금지).
- **터치 타깃**: 최소 44×44pt.
- **스크린리더**: VoiceOver/TalkBack 라벨(단계·번호·액션). device_id 등 민감값 노출 금지.
- **reduce motion**: 시스템 설정 시 애니메이션 축소/제거.

## 애니메이션
- fade-in, 짧은 slide만 허용. 글로우·바운스·연속 펄스 금지. reduce motion 존중.

## 아이콘
- 단순 라인 아이콘(strokeWidth 1.5). 둥근 배경 박스로 감싸지 않는다.
