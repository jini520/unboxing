# UI 디자인 가이드

> 대상: Expo (React Native). 웹 Tailwind가 아니라 RN `StyleSheet`/스타일 객체 기준.
> **테마는 시스템 외형(light/dark)을 따르며 라이트가 설계 기준**(→ ADR-016). 색은 하드코딩하지 말고 아래 **시맨틱 토큰**(라이트/다크 값)으로 쓴다.
> 색상·수치는 **초안** — Phase 1 UI 구현 step에서 확정한다. 원칙·안티패턴·접근성·상태/테마 규칙은 확정.
> 기능/플로우/스토어 정책은 `PRD.md`, 데이터/엔드포인트는 `ARCHITECTURE.md`.

## 디자인 원칙
1. **도구처럼 보여야 한다.** 마케팅이 아니라 매일 쓰는 배송 추적 대시보드. 신뢰감 있고 깔끔하게.
2. **시스템 테마를 따르고, 라이트를 기준으로 설계한다.** 대중 택배 앱의 표준 기대치는 라이트. 다크는 동등 품질로 지원(다크 전용/디폴트 아님).
3. **마찰 최소.** 등록은 두세 탭 안에. 불필요한 모달·온보딩 단계 금지.
4. **상태는 색에만 의존하지 않는다.** 색 + 아이콘 + 텍스트 라벨을 항상 함께(접근성).

| **등록** | 인앱 헤더(작은 title "운송장 번호 등록" + 자동채움 설명) → 번호 입력 → **(입력창 하단) 클립보드 제안** → 택배사 추정(**후보 1개면 자동 채움·여러 개면 자동선택 없이 드롭다운 명시 선택** — ADR-026) → 등록(택배사 미선택 시 비활성). 미지원이면 딥링크 카드 |
| **온보딩** | 푸시 권한 사전 안내(priming) → 시스템 요청 |
| **설정**(하단 탭) | 알림 토글 · 테마(시스템/라이트/다크) · **(v1.1) 시작 화면(택배함/대시보드, 기본 택배함)** · 개인정보처리방침(인앱 화면+웹 링크) · **모든 데이터 삭제**(메모·카테고리·금액·휴지통 포함) · 앱 버전 *(완료 숨기기 토글은 A2 정정으로 **택배함 상단**으로 이동)* |
| **빈 상태** | 가치 제안 한 줄 + 첫 등록 CTA (중앙 정렬 허용) |

- **네비게이션**: 하단 탭 **3개**(대시보드·택배함·설정 — **v1.1에서 대시보드 추가·좌측**). 상세/등록/온보딩/개인정보처리방침 **+ (v1.1) 알림 기록(`/notifications`)·휴지통(`/trash`)**은 탭 위로 push 되는 화면. **네이티브 헤더는 끄고**(headerShown:false) 각 화면이 공용 `ScreenHeader`(배경 없는 아이콘-only 뒤로가기 + 선택적 title/설명)를 그린다 — iOS 26 헤더 버튼 glass 배경 회피. **콜드스타트 초기 탭**은 "시작 화면" 로컬 preference(기본 택배함, ADR-025) — preference 미로드/실패 시 택배함 폴백.
- **택배함 탭 아이콘**: **닫힌 상자(Package)로 통일** — 활성/비활성 동일 글리프, 색만 토큰으로 구분(열린 상자 금지, 사용자 요구).
- **단계 인디케이터(상세)**: happy-path 5단계(등록→집화→이동중→배송출발→배송완료)를 가로 스텝퍼로. **각 단계 아이콘을 점 위에** 두고(점은 트랙에 유지), 현재=강조 색·크게, 지난=중립, 이후=비활성. off-track(미등록·기타=pre, 예외=경고)은 선형 매핑 금지.

## 테마 & 색상 (시맨틱 토큰 — 라이트 기준)

> 컴포넌트는 hex가 아니라 토큰명을 참조. 라이트가 기본, 다크는 시스템이 다크일 때.

### 배경 / 보더
| 토큰 | 라이트 | 다크 |
|------|------|------|
| `bg/page` | `#ffffff` | `#000000` |
| `bg/surface`(카드) | `#ffffff` | `#1c1c1e` |
| `bg/secondary` | `#f2f2f7` | `#2c2c2e` |
| `border` | `#e5e5ea` | `#38383a` |

- 라이트에선 페이지·카드가 모두 흰색이므로 카드는 **옅은 보더(+아주 약한 그림자)**로 구분. 다크에선 배경색 차이로 구분(그림자 금지).
- **값은 애플 iOS 시스템 컬러**: 배경=systemBackground/secondary·tertiarySystemGroupedBackground, `border`=systemGray5(L)·opaqueSeparator(D). 다크 페이지는 OLED true black(`#000000`). hex 단일 출처는 `app/src/theme/tokens.ts`(각 값 옆 Apple 컬러명 주석).

### 대표(브랜드) 색
| 토큰 | 라이트 | 다크 |
|------|------|------|
| `accent`(브랜드) | `#007aff` | `#0a84ff` |
| `onAccent`(accent 위 전경) | `#ffffff` | `#ffffff` |

- **대표 컬러 = 블루**. **상호작용 요소**에 쓴다(브랜드 강조): 기본 버튼·CTA·**활성 탭 틴트**·**선택 상태**(카드 보더·체크박스)·링크.
- **단계 색과 분리**(회귀 금지) — 과거 선택/링크가 `stage.outForDelivery`(배송출발 파랑)를 accent 대용으로 재사용하던 것을 `accent` 로 디커플링했다. `배송출발` 배지는 계속 `stage.outForDelivery` 사용(의미색). **선택/링크/버튼에 `stage.*` 를 직접 쓰지 말 것**.
- **토스트는 예외** — `text/primary` 배경(흑/백) 유지(브랜드색 아님, 사용자 요구).
- accent 위 텍스트/글리프는 **`onAccent`**(두 스킴 모두 흰색 — blue 위 대비 확보). 채워진 accent 버튼에 `bg/page` 를 전경으로 쓰지 말 것(다크에서 blue 위 대비 부족).

### 텍스트
| 토큰 | 라이트 | 다크 |
|------|------|------|
| `text/primary` | `#000000` | `#ffffff` |
| `text/body` | `#3c3c43` | `#ebebf5` |
| `text/secondary` | `#8e8e93` | `#8e8e93` |
| `text/disabled` | `#c7c7cc` | `#48484a` |

- 애플 label 램프: primary=label, body=secondaryLabel(불투명), secondary=systemGray, disabled=systemGray3. ⚠️ `text/secondary`(`#8e8e93`)는 흰 배경 대비 ≈3.5:1 로 **WCAG AA(4.5) 미달** — iOS 정통값이나, 캡션(상대시간·택배사명 12px)에 쓰이므로 대비가 걱정되면 더 진한 회색으로 조정 가능.

### 단계(시맨틱) 색 — 라이트/다크 모두 대비 확보
| 단계 | 라이트 | 다크 |
|------|------|------|
| `배송완료`(성공) | `#34c759` | `#30d158` |
| `배송출발`(임박) | `#007aff` | `#0a84ff` |
| `이동중`(진행 강조) | `#e0a500` | `#ffd60a` |
| `예외`(에러) | `#ff3b30` | `#ff453a` |
| `등록`·`집화`·`기타`(중립) | `#8e8e93` | `#8e8e93` |
| `미등록`(비활성) | `#aeaeb2` | `#636366` |

- 애플 시맨틱: 완료=systemGreen, 출발=systemBlue, 이동중=systemYellow, 예외=systemRed, 중립=systemGray, 미등록=systemGray2. `배송출발`이 `accent` 와 동색(둘 다 systemBlue)이나 토큰은 분리 유지(디커플링 규칙).
- **이동중 = 노란색 계열**(`stage.inTransit`, 사용자 요구) — 중간단계 중 유일하게 색 부여. ⚠️ 애플 systemYellow(`#ffcc00`)는 **흰 카드 위 대비 ≈1.5:1 로 라벨이 거의 안 보여**, 라이트는 톤 내린 골드(`#e0a500`, ≈2.2:1)로 둔다(다크는 systemYellow 정값 `#ffd60a`). `집화` 는 색을 바꾸지 않고(중립 유지) **아이콘만** 차별화.

## 단계 배지 (색 + 아이콘 + 라벨 — 색 단독 금지)

| 단계 | 아이콘(예) | 라벨 |
|---|---|---|
| `배송완료` | ✓ check(초록) | 배송완료 |
| `배송출발` | **truck(파랑)** | 배송출발 |
| `이동중` | **route(노랑)** | 이동중 |
| `예외` | alert(빨강) | 예외 |
| `집화` | box/Package(중립) | 집화 |
| `등록`·`기타` | dot(중립) | 각 단계명 |
| `미등록` | clock | 미등록 |

- `집화`·`이동중` 은 중간단계지만 식별성을 위해 고유 아이콘 부여(나머지 중간단계 `등록`·`기타` 는 중립 dot). 카드 배지 소스는 `STAGE_META`(StageBadge.tsx), 상세 스텝퍼는 별도 `STEP_ICONS`(StageProgress.tsx). **`배송출발`=truck·`이동중`=route 는 두 소스에서 동일**(통일·회귀 금지 — 과거 스텝퍼가 둘을 뒤바꿔 씀). `이동중` route 는 "경로를 따라 이동", truck(배송출발)·핀과 구분.

색은 위 "단계(시맨틱) 색"을 단일 출처로(ARCHITECTURE 상태 정규화 7단계와 1:1).

## 상태별 UI (로딩/빈/에러 — 누락 금지)

| 상태 | 표시 |
|---|---|
| 로딩(목록) | 스켈레톤 카드(좌측 정렬), 캐시 있으면 캐시 먼저 + 상단 새로고침 인디케이터 |
| 로딩(상세 타임라인) | 단계 배지는 즉시(캐시), 타임라인은 스켈레톤 |

| 조회 실패(상세 upstream) | 마지막 알려진 단계 + "타임라인을 못 불러왔어요, 다시 시도" |
| 미지원 택배사 | "자동 추적 미지원" + 택배사 조회 **딥링크** 버튼 |
| 등록 실패 | 인라인 에러(코드 노출 금지) + 재시도, 입력값 보존 |
| 알림 비활성(권한 거부) | 상단 작은 배너 "알림 꺼짐 — 켜기"(설정 유도) |

## 송장 카드 해부
```
[단계 배지]                                  [음소거 아이콘?] [상태 변경 후 상대시간]
메모(중단 · primary 색, 없으면 "{YYYYMMDD}에 등록한 상품")  ← 좌측정렬(가운데 정렬 X)
택배사명 · 운송장 전체번호 ················· [카테고리 칩]  ← 하단 한 줄(좌:12px / 우:칩)
```
- 좌측 정렬. 단계 배지=색+아이콘+짧은 라벨. 상대시간은 **상태 변경 시각**(`statusChangedAt`) 기준.
- **현재 상태는 StageBadge 로만 표시(회귀 금지)**: 단계 배지(색+글리프+라벨, 예 "이동중")가 곧 상태 표시다. **별도 "현재 상태 메세지" 텍스트 줄(STAGE_SUMMARY·STAGE_STATUS_MESSAGE)을 카드에 추가하지 말 것** — StageBadge 와 역할이 겹쳐 제거하기로 정함(사용자 요구). 위치(허브)는 목록 데이터에 없어 카드 표시 불가(상세 화면만 "이동 중 (허브명)"). 참고: `STAGE_SUMMARY` 상수 자체는 보존(상세 폴백 등 — 임의 제거 금지).
- **메모(중단 · 로컬 전용 · 미전송)** — tracker.delivery 에 상품명이 없어 사용자가 직접 적는 식별 정보. **중단 배치 · 좌측정렬**(가운데 **정렬** 금지 — 사용자가 말한 "가운데"는 **위치**[중단]이지 text-align 이 아니다). 색은 **primary**(default 문구라도 **회색 금지**). 없으면 등록일 기반 default(`{YYYYMMDD}에 등록한 상품` — 예 "20260619에 등록한 상품") — placeholder 가 아니라 **대체 문구**(항상 한 줄). 카드와 **상세 페이지 타이틀**이 같은 문구를 쓰도록 `defaultMemoText()`(memo.ts) 단일 출처. 편집은 상세 헤더 연필 → 모달. 저장은 로컬 AsyncStorage.
- **택배사 · 운송장 전체번호(하단)** — 보조 정보라 **하단 · 작은 폰트(12)**. 운송장 번호는 **전체** 표기(본인 데이터, 끝4자리 X), 길면 잘림 없이 줄바꿈 허용.
- **카테고리 칩 = 하단 행 우측(v1.1.3 · ADR-041 · 회귀 락)** — 카테고리 칩은 **택배사·번호 줄과 같은 행의 우측 끝**에 둔다(독립 줄 아님). 하단 행 = `flexDirection:"row", alignItems:"flex-end", justifyContent:"space-between"` — 좌측 택배사·번호(`flex:1`·길면 줄바꿈), 우측 칩(`flexShrink:0`·하단 정렬). 칩 미설정이면 우측 비움. **카드는 3줄(배지·메모·하단행) 고정**(칩 유무 무관, 과거 칩이 독립 줄이라 4줄로 늘던 것 해소). **회귀 금지**: 칩을 다시 독립 줄로 빼거나 메모 줄에 붙이지 말 것. 메모 표시 규칙(중단·좌측·`primary`·대체문구·1줄)은 불변(칩은 보조).
- **양방향 스와이프 — 2단계 + 넓은 버튼**: **좌측 스와이프=삭제**, **우측 스와이프=음소거**.
  - 1단계(첫 스와이프): 버튼을 **노출(rest)만** — 길게 스와이프해도 실행되지 않는다. 버튼 **탭**으로도 실행.
  - 2단계: **노출된 상태에서 같은 방향으로 추가로 `SWIPE_AGAIN_PX`(기본 80px)만큼 더 밀면** 버튼 탭과 동일 실행. **민감도는 이 상수 하나로 조절**(클수록 둔감). `openedRef`(onSwipeableOpen/Close)로 "이미 열림"을 게이트해 **첫 스와이프는 실행 안 됨**.
  - **2단계 임계 도달 시 햅틱**(양방향=삭제·음소거, 사용자 요구): dragX 리스너가 예약(`pendingFireRef`)하는 그 순간 `Haptics.impactAsync(Medium)`로 한 번 — 임계를 넘었다는 즉각 피드백(실행은 정착 시점). iOS 시뮬레이터/햅틱 비활성 기기에선 no-op(throw 무시), reduce-motion 무관(시스템 햅틱 설정이 별도 게이트라 expo-haptics 가 자동 존중). `expo-haptics` 의존성 추가됨(SDK 56).
  - **CRITICAL(회귀 금지) — 구현 방식**: 거리 측정은 반드시 **base `AnimatedValue` 인 `dragX`**(제스처 raw translationX, swipeable 내부 `state.dragX`)에 `addListener`. render 콜백이 주는 `progress`(1번째)·`transX`(2번째)는 **interpolation 이라 RN Animated 가 리스너를 발화시키지 않는다**(base Value 만 `__callListeners` 호출). 또 Swipeable 기본 `useNativeAnimations:true`(native driver)면 JS 리스너가 아예 안 오므로 **`useNativeAnimations={false}` 필수**. `onSwipeableWillOpen`-재열림 단독 트리거는 거리 임계가 없어 **과민**(회귀).
  - **CRITICAL(회귀 금지) — 실행 타이밍**: 임계 도달 시 **제스처 진행 중에 액션/`close()`를 호출하면 안 된다** — 진행 중 pan 과 close 애니메이션이 충돌해 **카드가 떨리고**, 손을 떼면 `handleRelease` 가 다시 열어 **1단계에 잔류**한다. dragX 리스너는 임계 도달 시 **예약(pendingFireRef)만** 하고, **정착 시점 `onSwipeableWillOpen` 에서 실행+`close()`**. 구현 `app/src/components/ShipmentCard.tsx`.
  - 참고: 이 `Swipeable`(react-native-gesture-handler legacy)는 **deprecated**(Reanimated 버전 권장)이나 동작엔 문제 없음. 마이그레이션은 `react-native-reanimated` 신규 의존 추가 + 스와이프 재작성이 필요해 **별도 작업**(미설치 상태).
  - **과스와이프 제한(회귀 금지)**: rest(버튼 노출) 너머에서 RNGH `transX` 는 clamp 가 아니라 `1/overshootFriction` 기울기로 **연장**되므로, 기본값 1 이면 카드가 버튼 폭 너머로 끝없이 따라온다(사용자 보고 "너무 많이 스와이프됨"). **`overshootFriction={OVERSHOOT_FRICTION}`(=8)** 로 둔화해 카드를 버튼 폭 근처에 거의 고정한다(라이브러리 권장 "8 or above"). ⚠️ 이 값은 **시각 `transX` 에만** 작용하고 2단계 판정 기준 raw `dragX` 엔 영향 없어 `SWIPE_AGAIN_PX` 는 그대로 동작 — **overshoot 자체를 끄는 방식으로 되돌리지 말 것**(2단계 회귀). 구현 `app/src/components/ShipmentCard.tsx`.
  - **CRITICAL(회귀 금지) — 배경 폭**: 컬러 배경은 노출 가장자리에 앵커한 **한정 폭**(`ACTION_BG_WIDTH`≈120, 음소거=좌측 `actionBgLeft`·삭제=우측 `actionBgRight`)로 깐다. 콘텐츠(아이콘+라벨)는 고정폭(`actionBtn` 96, 스냅 폭). **`StyleSheet.absoluteFill`(카드 폭 전체) 풀폭으로 되돌리지 말 것** — RNGH 패널이 컨테이너 전폭(absoluteFill)이라, 풀폭 배경 + 컨테이너 `overflow:visible`(카드가 거터 넘어 끝까지 밀리게 한 설정)이 만나면 **토글 실행 후 카드 복귀 전환 때 반대편 모서리로 컬러가 샌다**(사용자 보고). `transX` 가 `leftWidth(≈버튼폭)+overshoot(~1px)` 로 클램프돼 노출 폭이 버튼 폭을 안 넘으므로 한정 폭이면 과스와이프 틈도 없고 far edge 가 항상 카드 아래라 안 샌다. **고정폭 배경 자체는 유지**(콘텐츠 96 shrink/clip 금지)·overshoot 유지. 구현: `app/src/components/ShipmentCard.tsx`.
  - **버튼 색/아이콘(회귀 금지)**: **음소거=`accent`(브랜드 블루) 배경**, **삭제=`stage.exception`(빨강) 배경**, 두 버튼 모두 글리프·라벨은 **`onAccent`(흰색)**. 아이콘은 **채움(fill) 글리프**(`BellFill`/`BellOffFill`/`TrashFill` — stroke 라인 버전 아님, 사용자 요구). 카드의 음소거 표시(작은 14px)는 그대로 라인 `BellOff`. **중립 `bg/secondary` 배경 / stroke 아이콘으로 되돌리지 말 것**.
  - **삭제는 확인 다이얼로그**("삭제할까요?" + 식별 문구[메모/대체문구] → 취소/삭제) — **Undo 토스트는 폐기**(회귀 금지, 사용자 요구. 과거 "Undo로 복구·확인 없음"에서 전환). 음소거는 낙관 토글+실패 시 롤백(토스트 안내). 탭 → 상세.
- **롱프레스 → 멀티선택**: 체크박스 표시·탭=선택 토글(상세 이동 X)·스와이프 비활성.
  - 헤더 1행: **[취소] · {N}개 선택(가운데) · [일괄삭제]** — 카운트는 화면 중앙(양 끝 동일 폭).
  - 헤더 2행(아래): **전체 선택 / 전체 해제**(전체 해제는 선택만 비우고 선택 모드 유지). 일괄삭제=확인 다이얼로그·**Undo 없음**·부분 실패 복원. 0개 선택이어도 취소/뒤로 전엔 유지.
  - **전체 선택·전체 해제 스타일(회귀 금지)**: **취소 버튼과 동일한 중립 회색**(`text/secondary`)·**emphasis 없음**(굵게/강조색 금지) — 보조 액션이라 눈에 덜 띄게(사용자 요구).
- 음소거 카드는 한 켠에 음소거 아이콘(색 단독 아님, a11y "알림 꺼짐").
- 택배사 식별: 택배사명 + (선택) 택배사 색 점/로고 — 로고 사용 시 **상표 가이드 준수**(임의 변형·왜곡 금지, 미확보 시 텍스트/중립 아이콘으로 대체).

- 탭 동작: 진행 중→**택배함 탭**, 배송완료→**택배함 탭**(필터 프리셋 **없음** — A2 로 필터 제거), 휴지통→`/trash`, 새 알림→`/notifications`.
- (제안) **이번 달 등록 금액 합계** 카드: 로컬 금액 합(₩ 포맷). 일부만 입력 시 "일부 미입력" 캡션. 미입력 0건이면 카드 숨김 가능.
- 빈 목록(송장 0)이어도 **4카드를 그대로 표시**(카운트 대부분 0 — 단 휴지통·새 알림은 0이 아닐 수 있음). **대시보드엔 등록 CTA 빈화면을 두지 않는다**(2026-06-23 #1 정정 — 현황판이라 0이라도 카드 표시. 로딩 중에만 스피너). 택배함(목록) 빈 상태 CTA는 그대로 유지. 오프라인: 캐시 기반 집계 + 신선도.

### 헤더 알림 종 + 미읽음 배지
- 택배함·대시보드 헤더 우측 종(line `Bell`). 미읽음>0이면 우상단 작은 점/숫자 배지(`accent` 배경·`onAccent`). 0이면 배지 없음.
- 탭 → `/notifications`. a11y 라벨 "알림(미읽음 N개)". 터치 타깃 ≥44.

### 알림 기록 화면 (`/notifications`)
- `ScreenHeader`(뒤로 + title "알림"). 시간 역순 리스트(날짜 구분 헤더 선택).
- 항목: 좌측 단계 아이콘(StageBadge 글리프 재사용·의미색) + **택배사 한글명 · 상태 문구**(본문) + 우측 상대시간. 미읽음은 좌측 점 또는 굵게(색 단독 금지).
- 탭 → 해당 상세 딥링크. 대상이 정리됨(`shipment_id` null/404)이면 비활성 + 토스트 "정리된 택배예요".
- 빈 상태: 알림 권한 꺼짐이면 "알림이 꺼져 있어요 — 켜면 배송 알림이 여기 쌓여요"(설정 유도), 켜짐인데 없으면 "받은 알림이 없어요". 열람 시 `lastSeen` 갱신 → 배지 0(첫 실행은 `now` 초기화라 기존 기록이 미읽음으로 폭주하지 않음 — ARCHITECTURE 보강 ⑤). 오프라인은 로컬 캐시 표시.

### 휴지통 화면 (`/trash`)
- `ScreenHeader`(뒤로 + title "휴지통" + 선택적 "일괄" 액션). 항목 = 송장 카드 축약(단계 배지 + 메모/대체문구 + 택배사·번호) + **"N일 후 영구 삭제"** 캡션(`text/secondary`, 임박 시 `예외` 색).
- 행 액션: **복구**(`accent` 텍스트/버튼) · **영구 삭제**(`예외` 색·확인 다이얼로그). 멀티선택 시 **일괄 복구 / 일괄 영구 삭제**(영구 삭제는 확인 다이얼로그·Undo 없음 — 택배함 일괄삭제와 동일 규칙).
- 복구 실패(429/409)는 항목 유지 + 안내. 빈 상태: "휴지통이 비었어요".

### 운송장 수정 (택배사·번호) — 상세 헤더 연필 `Pencil` (2026-06-23 #2·#4)
- 상세 헤더 **연필("수정")** → 모달에서 **택배사·운송장번호** 편집. 택배사 선택은 **등록 화면과 동일 컴포넌트·정책 재사용**(후보 ≥2 면 자동선택 없이 드롭다운 명시 선택 — ADR-026). 저장 버튼은 택배사 미선택·번호 무효 시 비활성.
- **저장 = 재등록**(ADR-027, 서버 행 직접 수정 아님): `POST /shipments(새 택배사·번호)`[먼저] → 택배 정보(메모·카테고리·금액) old→new id 이관 → `DELETE` 기존 → 새 상세로 교체. **변경 없음=no-op**. 새 조합이 **이미 추적 중**이면 "이미 추적 중인 운송장" 안내 + 그 상세로 이동(중복 구독 안 만듦).
- 안내 카피: 식별자 변경은 "사실 다른 택배"라 **이전 타임라인은 새 번호로 새로 시작**됨을 모달에 한 줄 고지(예: "번호를 바꾸면 새 운송장으로 다시 추적해요"). 코드·기술 메시지 비노출(PRD 톤).
- **회귀 금지**: 삭제 버튼·타임라인 고정 스크롤·메모 인라인 박스 없음 규칙 등 기존 상세 인터랙션 불변.

### 택배 정보 모달 (상세 헤더 `FileText` 아이콘 → 모달 — 메모/카테고리/금액)
> 2026-06-23 #2: 상세 헤더의 단일 연필이 열던 모달을 **두 진입점으로 분리**. 이 "택배 정보"(메모·카테고리·금액·로컬)는 **연필("수정")과 별개의 `FileText`(신규 글리프) 아이콘**으로 연다. 택배사·번호 편집은 위 "운송장 수정" 섹션 참조. 두 아이콘은 헤더 우측에 나란히(터치 타깃 ≥44, a11y 라벨 각각 "수정"·"택배 정보").
- 메모 편집 모달을 **메모 + 카테고리 + 금액** 3필드로(인라인 박스 없음·헤더 아이콘 진입 규칙 유지 — 회귀 금지).
- **메모**: 기존 textarea(모달 내). **카테고리**: 고정 목록 선택(칩 또는 셀렉트). **선택 안 함이 기본(미설정=값 없음·칩 없음)** — 목록 마지막 `기타`는 실제 catch-all 카테고리이지 "미설정"이 아니다(혼동 금지). **금액**: 숫자 키패드 입력 + ₩ 접두/천단위 구분 표시, **0 이상 정수만**(음수·비정수·상한 초과는 인라인 안내·저장 안 함).
- 저장 = 로컬 스토어(`unboxing.shipment_info`). 빈 메모는 기존대로 삭제. 카테고리 미설정·금액 빈값은 해당 필드 미저장.
- **카드 카테고리 칩**(v1.1.3 확정): 송장 카드 **하단 행(택배사·번호) 우측 끝**의 작은 칩(중립 `bg/secondary`·`Tag` 글리프). 위치 규칙·회귀 락은 위 "송장 카드 해부 → 카테고리 칩" 단일 출처(ADR-041). **메모 표시 규칙(중단·좌측·primary·대체문구)은 불변**(회귀 금지) — 칩은 보조.
- **(v1.1.1) 모달 동작(택배 정보·운송장 수정 공통)**: 바깥(backdrop) 탭으로 **닫히지 않음**(바깥 탭=키보드만 접기), 닫기는 취소/뒤로만. 키보드 활성 시 `KeyboardAvoidingView` 로 가용 영역 재중앙(가림 방지). 상세 규칙·회귀 락 → 위 "v1.1.1" 섹션·ADR-034·ENGINEERING P-9.

### 택배함 완료 숨기기 토글 (목록 위) — 2026-06-23 A2 정정
> 필터 칩(전체/진행 중/임박/완료/예외)은 **폐기**(보류) — 세로 stretch 레이아웃 깨짐도 칩 제거로 자동 해소. 대신 "배송 완료된 항목 감추기" 토글만 두고, 위치를 설정에서 택배함으로 옮겼다.

- 멀티선택 모드와 공존(보이는 목록 기준 전체선택). 대시보드 카드 탭은 **필터 프리셋 없이** 택배함 탭으로만 진입(A2).

### v1.1.1 — 수취인 표시 게이트·헤더 고정·모달 동작 (2026-06-30 · 회귀 락)
- **수취인 "받는 분" 줄(상세 상단 우측)** — 이름이 **식별 가능할 때만** 표시. 앱 순수 함수 `displayRecipientName`(ADR-032)이 ① 빈/공백 ② 플레이스홀더(`받는 분`·`받는분`·`수령인`·`수취인`·`수신인`·`고객(님)`·`본인` 등) ③ 마스킹문자 제거 후 문자 0개(`***`)를 **숨김**, 부분 마스킹(`김**`·`이**`·`김*윤`)·실명은 표시한다. **회귀 금지**: `recipient?.name` 을 게이트 없이 바로 렌더하지 말 것(택배사가 라벨을 이름으로 보내 `받는 분 받는 분` 중복 표기됨). 수취인은 여전히 **미저장 패스스루**(ADR-005) — 게이트는 표시 전용.
- **페이지 제목 고정(전 화면)** — 제목(헤더)은 **스크롤 영역 밖 고정**(스크롤돼 사라지지 않음). 택배함·대시보드·알림·상세·휴지통·등록은 이미 고정, **설정만 `ScrollView` 안에 있던 제목/설명을 고정 영역으로 분리**(택배함 패턴). 개인정보처리방침은 본문 제목 스크롤 허용(back 헤더는 고정). **회귀 금지**: 설정 제목/설명을 다시 `ScrollView` 안에 넣지 말 것. (ADR-033)
- **입력 모달 동작(택배 정보·운송장 수정)** — ① **바깥(backdrop) 탭으로 닫지 않음**: 바깥 탭은 `Keyboard.dismiss()` 만, 모달 닫기는 **취소 버튼·Android 뒤로**만. ② **키보드 회피**: `KeyboardAvoidingView`(iOS `padding`/Android `height`)로 키보드 위 가용 영역에서 재중앙 정렬. **회귀 금지**: backdrop `onPress={() => setModal(false)}` 로 되돌리지 말 것(오터치 닫힘 재발). iOS `<Modal>` 내 `KeyboardAvoidingView` 함정·스모크 → ENGINEERING P-9. ### v1.1.2 — 캡처로 채우기 (택배 정보 모달 · 자동 채움 보조)
- **진입·병행**: "택배 정보" 모달 상단(또는 메모 필드 옆)에 **"캡처로 채우기"** 버튼. 기존 직접 입력(메모·카테고리 칩·금액)은 **그대로 유지** — 캡처는 보조다(ADR-039). **회귀 금지**: 직접 입력 경로를 제거하거나 캡처를 강제 단계로 만들지 말 것.
- **흐름**: 버튼 → 이미지 선택(`expo-image-picker`) → 온디바이스 OCR → 마스킹 → 분류 → 모달의 메모·금액·카테고리 칩 **자동 채움** → 사용자가 기존 컴포넌트로 교정 → 기존 저장.
- **진행 표시 (v1.1.3 · ADR-045)**: 분석 중 모달 카드 위에 **반투명 오버레이 + 중앙 스피너 + 단계 텍스트** — ① 온디바이스 OCR 구간 `이미지 인식 중…` → ② Workers AI 분류 구간 `상품 분석 중…`. 분류 엔드포인트가 상품·가격·카테고리를 **1콜로 반환**(ADR-037)하므로 필드별 부분 채움이 아니라 **분석 완료 후 매핑 필드 일괄 자동 채움**(편집 가능한 **초안** — 확정 아님). 오버레이는 분석 중 필드 오터치 차단. **회귀 금지**: 가짜 단계로 쪼개지 말 것(OCR/분류 경계만)·오버레이가 폴백 `Alert`(아래)을 가리지 말 것.
- **폴백(누락 금지)**: OCR 0줄 → "인식 실패, 다시 촬영". 분류 5초+/한도초과/JSON 깨짐 → 카테고리 "미분류" + 수동 선택. **어느 단계가 죽어도 직접 입력·저장 흐름은 안 멈춤**.
- 색·컴포넌트는 토큰만(아래 프리미티브). 광고성 아님(ADR-018 무관). 결정·파이프라인 → ADR-036~039 · ARCHITECTURE "v1.1.2".

  - **저장 버튼 = 채움형**: 하단 액션을 텍스트-only 에서 → **취소(텍스트·`text/secondary`) + 채움형 저장 버튼**(`accent` 배경·`onAccent` 라벨·`radius.md`·우측). 비활성(금액 무효 등) 시 `bg/secondary`+`text/disabled`. 저장 색은 **`accent` 고정**(stage.* 직접 사용 금지 — 디커플링 규칙).
  - **타이포/간격**: 제목 `title3`/`semibold`, 카드 패딩·필드 간격은 프리미티브로 통일.
  - **회귀 금지**: 바깥 탭 닫힘 복원 금지(ADR-034)·**바텀시트 전환 금지**(중앙 카드 유지 — 키보드 회피 P-9 검증 보존)·저장 버튼 그라데이션/네온 금지.
- **송장 등록 FAB(대시보드·택배함 · #3 · ADR-042)** — 화면 **우하단 원형 FAB** → 탭 시 `/register`. 디자인: 원형 **지름 ≈56**, `accent` 배경, `onAccent`(흰) `Plus` 글리프(기존 아이콘), **라이트만 옅은 그림자**(다크 그림자 금지). 위치 `position:"absolute"`, `right:spacing.lg` · `bottom:` = 하단 inset(탭바·홈 인디케이터) + `spacing.lg`. 공용 컴포넌트 `Fab.tsx`. a11y "운송장 등록"·터치 ≥44.
  - **노출 조건**: 송장 있고 **비선택 모드일 때만**(택배함 빈 상태=중앙 등록 CTA 존재·멀티선택 모드=액션 헤더 충돌 → **숨김**). 대시보드는 항상 노출(현황판). 리스트 하단 토스트와 z-순서/위치 겹침 주의.
  - **택배함 헤더 우상단 기존 '+' 버튼은 유지**(둘 다 유지 — 사용자 결정). 색은 `accent` 토큰만(스플래시/아이콘 하드코딩 `#1b68eb`와 다름 — UI accent = `#007aff`).
- **등록 후 "택배 정보 입력" 확인(#4 · ADR-043)** — 운송장 등록 성공 직후 **native `Alert`** "택배 정보를 입력할까요?"(삭제 확인과 동일 톤). **확인** → 새 상세로 이동 + "택배 정보" 모달 자동 오픈(`router.replace({pathname:"/shipment/[id]", params:{id, openInfo:"1"}})`, 상세가 `openInfo` 읽어 1회 오픈). **취소** → 기존 흐름(목록/뒤로). **첫 등록 푸시 priming(`/onboarding`)이 우선** — priming 대상이면 확인 없이 온보딩. **멱등 등록(이미 추적 중)**: 확인 스킵·그 상세로. 카피는 친근체(코드 비노출 — 마이크로카피 톤).
- **저장값 prefill (#7 · ADR-046)** — "택배 정보" 모달은 **모든 열기 경로**(헤더 `FileText`·#4 자동오픈)에서 이미 저장된 메모·카테고리·금액을 **입력 필드에 채운 채** 연다. 저장값 로드(`getInfo`) 시 draft 도 미러(비동기 타이밍 안전). 신규 송장은 빈 칸(정상). **회귀 금지**: 헤더 재오픈 시 저장값 재-prefill(미저장 편집 폐기) 유지·캡처 자동채움(ADR-045) 덮어쓰기 불변.

## 디자인 프리미티브 (간격·라운드·타이포 — 단일 출처)

> 스킴(light/dark) **무관** 공통 수치. 색은 위 시맨틱 토큰, 이 값들은 `app/src/theme/layout.ts` 에 모은다. 컴포넌트는 StyleSheet 리터럴 대신 이 토큰을 참조한다(공통 값 단일 출처). 스케일에 없는 1회성 값(원형 점 반지름=width/2·스와이프 버튼 폭·아이콘 `size`·`lineHeight` 등)은 컴포넌트 로컬 리터럴로 둔다.

- **spacing**(padding·margin·gap — 4단위): `xs 4 · sm 8 · md 12 · lg 16 · xl 24 · xxl 32`. 페이지 좌우 거터=`lg`(16), 섹션 간격=`xl`~`xxl`.
- **radius**(모서리): `sm 6`(체크박스) · `md 8`(카드·버튼·입력·배너·토스트) · `lg 12`(모달).
- **fontSize**(타입 램프): `micro 11 · caption 12 · footnote 13 · body 14 · callout 15 · base 16 · title3 17 · title2 19 · title1 22 · display3 24 · display2 28 · display1 30`.
- **fontWeight**: `medium 500 · semibold 600 · bold 700`.

## 컴포넌트 (초안 — 토큰 참조)
### 카드 (송장 항목)
```
borderRadius: 8, backgroundColor: bg/surface, borderWidth: 1, borderColor: border, padding: 16
(라이트: 옅은 그림자 1단계만 / 다크: 그림자 없음)
```
### 버튼
```
Primary: backgroundColor accent(브랜드), 라벨 onAccent(흰색), borderRadius radius.md(8)
Text:    color text/secondary (press → text/body)
Destructive: color = 단계 '예외' 색
Link:    color accent(브랜드)
```
### 입력 필드 (운송장 번호)
```
backgroundColor bg/secondary, borderWidth 1, borderColor border, borderRadius 8, paddingHorizontal 16, paddingVertical 12
```

## 레이아웃
- 정렬: 좌측 정렬 기본. 중앙 정렬은 빈 상태에서만.
- 간격: 항목 간 8~12, 섹션 간 24~32. (→ `spacing` 토큰: `sm`~`md`, `xl`~`xxl`)
- 리스트가 주 화면 — 카드 스택 + 당겨서 새로고침. **정렬**: 진행 중 우선, 임박·예외 강조.

## 인터랙션
- **당겨서 새로고침**: 목록·상세. 오프라인이면 안내.
- **양방향 스와이프(버튼 노출 → 탭/추가 스와이프 실행)**: **좌측 스와이프=삭제**(reveal → **확인 다이얼로그**로 실수 방지, Undo 토스트 폐기·회귀 금지), **우측 스와이프=음소거**(낙관 토글 + 실패 시 롤백). 첫 스와이프는 버튼만 노출(정지). **2단계 임계 도달 시 햅틱**(양방향).
- **롱프레스 멀티선택 + 일괄삭제**: 롱프레스로 선택 모드 진입(체크박스·전체선택·취소). 일괄삭제는 **확인 다이얼로그 + Undo 없음**(부분 실패 시 해당 항목 복원·안내). **단건·일괄 삭제 모두 확인 다이얼로그로 통일**(단건 Undo 폐기).
- **클립보드 제안**: 등록 화면 진입 등 명시 시점에만, 제안만(자동 등록 금지 — iOS 배너 정책, PRD 권한). 위치는 **운송장 번호 입력창 하단**(상단 아님 — 사용자 요구).
- **토스트/에러**: 일시 오류는 비차단 토스트, 영구 오류는 인라인.
