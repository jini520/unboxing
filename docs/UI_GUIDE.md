# UI 디자인 가이드

> 대상: Expo (React Native). 웹 Tailwind가 아니라 RN `StyleSheet`/스타일 객체 기준.
> **테마는 시스템 외형(light/dark)을 따르며 라이트가 설계 기준**(→ ADR-016). 색은 하드코딩하지 말고 아래 **시맨틱 토큰**(라이트/다크 값)으로 쓴다.
> 색상·수치는 **초안** — Phase 1 UI 구현 step에서 확정한다. 원칙·안티패턴·접근성·상태/테마 규칙은 확정.
> 기능/플로우/스토어 정책은 `PRD.md`, 데이터/엔드포인트는 `ARCHITECTURE.md`.

## 디자인 원칙
1. **도구처럼 보여야 한다.** 마케팅이 아니라 매일 쓰는 배송 추적 대시보드. 신뢰감 있고 깔끔하게.
2. **시스템 테마를 따르고, 라이트를 기준으로 설계한다.** 대중 택배 앱의 표준 기대치는 라이트. 다크는 동등 품질로 지원(다크 전용/디폴트 아님).
3. **마찰 최소.** 등록은 두세 탭 안에. 불필요한 모달·온보딩 단계 금지.
4. **상태는 색에만 의존하지 않는다.** 색 + 아이콘 + 텍스트 라벨을 항상 함께(접근성).

## AI 슬롭 안티패턴 — 하지 마라
(RN 맥락으로 적응. CSS 용어는 RN 대응으로 읽을 것.)

| 금지 사항 | 이유 |

## 화면 구성

| 화면 | 내용 |
|---|---|
| **택배함**(주화면·하단 탭) | 제목 **"택배함"** + 페이지 설명(마지막 업데이트 문구 아님) + ＋버튼(등록). 마지막 업데이트(신선도)는 **첫 카드 위 오른쪽**. 송장 카드 스택(메모 포함) + 당겨서 새로고침. 멀티선택 모드 헤더 |
| **상세** | ①택배사·전체번호(좌)+받는분(우) 한 줄 → ②현재 상태 문구(상단 중앙·크게·친절 교정, '상품' 통일) → ③단계 인디케이터(점 **위에** 단계 아이콘·크게) → ④메모(읽기 전용 박스, **헤더 연필/박스 탭 → 모달 편집**) → ⑤타임라인(실시간, 좌우 패딩) → 삭제 |
| **등록** | 인앱 헤더(작은 title "운송장 번호 등록" + 자동채움 설명) → 번호 입력 → 택배사 자동추정/확인 → 등록. 미지원이면 딥링크 카드 |
| **온보딩** | 푸시 권한 사전 안내(priming) → 시스템 요청 |
| **설정**(하단 탭) | 알림 토글 · 테마(시스템/라이트/다크) · 개인정보처리방침(인앱 화면+웹 링크) · **모든 데이터 삭제**(메모 포함) · 앱 버전 |
| **빈 상태** | 가치 제안 한 줄 + 첫 등록 CTA (중앙 정렬 허용) |

- **네비게이션**: 하단 탭 2개(택배함·설정). 상세/등록/온보딩/개인정보처리방침은 탭 위로 push 되는 화면. **네이티브 헤더는 끄고**(headerShown:false) 각 화면이 공용 `ScreenHeader`(배경 없는 아이콘-only 뒤로가기 + 선택적 title/설명)를 그린다 — iOS 26 헤더 버튼 glass 배경 회피.
- **택배함 탭 아이콘**: 활성=열린 상자(PackageOpen)·비활성=닫힌 상자(Package).
- **단계 인디케이터(상세)**: happy-path 5단계(등록→집화→이동중→배송출발→배송완료)를 가로 스텝퍼로. **각 단계 아이콘을 점 위에** 두고(점은 트랙에 유지), 현재=강조 색·크게, 지난=중립, 이후=비활성. off-track(미등록·기타=pre, 예외=경고)은 선형 매핑 금지.

## 송장 카드 해부
```
[단계 배지]  [음소거 아이콘?] [상태 변경 후 상대시간]
택배사명 · 운송장 전체번호
메모(있으면 메모, 없으면 "{등록일}에 등록한 택배")
```
- 좌측 정렬. 단계 배지=색+아이콘+짧은 라벨. 운송장 번호는 **전체** 표기(본인 데이터, 끝4자리 X). 상대시간은 **상태 변경 시각**(`statusChangedAt`) 기준.
- **메모(로컬 전용·미전송)** — 카드 셋째 줄. tracker.delivery 에 상품명이 없어 사용자가 직접 적는다. **메모는 택배사·번호보다 중요한 식별 정보**라 가장 읽기 쉬운 본문 색. 없으면 등록일 기반 default(`{M월 D일}에 등록한 택배`, 회색) — placeholder 가 아니라 **대체 문구**(항상 한 줄, UI 통일감). 편집은 상세 화면(헤더 연필 → 모달). 저장은 로컬 AsyncStorage(서버 미전송).
- **양방향 스와이프 — 2단계 + 넓은 버튼**: **좌측 스와이프=삭제**, **우측 스와이프=음소거**.
  - 1단계(가볍게 스와이프): 버튼을 **노출(rest)**. 버튼 **탭**으로 실행.
  - 2단계(끝까지 더 스와이프, `FULL_SWIPE` 임계 초과): dom 이 넘어가며 **바로 실행**.
  - **CRITICAL(회귀 금지)**: 뒤에 깔리는 알림/삭제 버튼은 **넓어야** 한다 — 배경을 `StyleSheet.absoluteFill` 로 액션 컨테이너(카드 폭 전체)에 깔아 과스와이프해도 카드-버튼 사이 **빈 틈이 없다**. 콘텐츠(아이콘+라벨)만 고정폭(스냅). **고정폭 배경 / overshoot 비활성으로 되돌리지 말 것**(틈/잘림 재발). 구현: `app/src/components/ShipmentCard.tsx`.
  - 삭제는 Undo 토스트로 복구(확인 다이얼로그 없음), 음소거는 낙관 토글+실패 시 롤백. 탭 → 상세.
- **롱프레스 → 멀티선택**: 체크박스 표시·탭=선택 토글(상세 이동 X)·스와이프 비활성.
  - 헤더 1행: **[취소] · {N}개 선택(가운데) · [일괄삭제]** — 카운트는 화면 중앙(양 끝 동일 폭).
  - 헤더 2행(아래): **전체 선택 / 전체 해제**(전체 해제는 선택만 비우고 선택 모드 유지). 일괄삭제=확인 다이얼로그·**Undo 없음**·부분 실패 복원. 0개 선택이어도 취소/뒤로 전엔 유지.
- 음소거 카드는 한 켠에 음소거 아이콘(색 단독 아님, a11y "알림 꺼짐").
- 택배사 식별: 택배사명 + (선택) 택배사 색 점/로고 — 로고 사용 시 **상표 가이드 준수**(임의 변형·왜곡 금지, 미확보 시 텍스트/중립 아이콘으로 대체).

## 타임라인 컴포넌트
- 세로 단계 점(현재 단계 강조) + 시각(KST, 상대+절대) + 설명/허브명(위치는 비개인정보).
- 최신이 위. 단계 전환점 라벨 강조.

## 알림 UI
- 단계별 카피는 `PRD.md` 알림 정책 참조. 본문 ≤ payload 4096B.
- **그룹화**: 여러 송장 알림은 묶음/요약(과알림 방지, 조용시간과 결합).
- Android: "배송 상태" notification channel 분리.
- 알림 탭 → 해당 **상세 딥링크**(payload `shipment_id`).

## 설정 / About 화면
- 항목: 알림 켜기/끄기, 테마(시스템·라이트·다크), 개인정보처리방침(외부 링크), **모든 데이터 삭제**(빨강·확인 다이얼로그·복구 불가 명시 → `DELETE /me`), 앱 버전.
- "모든 데이터 삭제"는 파괴적 액션 스타일(`예외` 색), 실행 후 결과 토스트.

```

```


```
backgroundColor bg/secondary, borderWidth 1, borderColor border, borderRadius 8, paddingHorizontal 16, paddingVertical 12
```

## 레이아웃
- 정렬: 좌측 정렬 기본. 중앙 정렬은 빈 상태에서만.
- 간격: 항목 간 8~12, 섹션 간 24~32.
- 리스트가 주 화면 — 카드 스택 + 당겨서 새로고침. **정렬**: 진행 중 우선, 임박·예외 강조.

## 마이크로카피 / 톤
- 친근한 한국어. **에러 코드/기술 용어 노출 금지**(내부 code는 로그만).
- 상대 시간("방금 업데이트", "2시간 전") + 절대 시각 보조.
- 실패 카피 예: "잠시 후 다시 시도해 주세요". 빈 상태: "운송장만 넣어두면 상태가 바뀔 때 알려드려요".

## 인터랙션
- **당겨서 새로고침**: 목록·상세. 오프라인이면 안내.
- **양방향 스와이프(버튼 노출 → 탭/추가 스와이프 실행)**: **좌측 스와이프=삭제**(reveal + Undo 토스트로 실수 복구, 서버 반영은 Undo 만료 후 — 별도 확인 다이얼로그 없음), **우측 스와이프=음소거**(낙관 토글 + 실패 시 롤백). 첫 스와이프는 버튼만 노출(정지).
- **롱프레스 멀티선택 + 일괄삭제**: 롱프레스로 선택 모드 진입(체크박스·전체선택·취소). 일괄삭제는 **확인 다이얼로그 + Undo 없음**(부분 실패 시 해당 항목 복원·안내). 단건 스와이프 삭제만 Undo 유지.
- **클립보드 제안**: 등록 화면 진입 등 명시 시점에만, 제안만(자동 등록 금지 — iOS 배너 정책, PRD 권한).
- **토스트/에러**: 일시 오류는 비차단 토스트, 영구 오류는 인라인.

## 접근성 (확정)
- **색 단독 금지**: 상태=색+아이콘+텍스트.
- **대비**: 라이트·다크 **양쪽 모두** 본문/배경 WCAG AA 이상(위 토큰은 이를 만족하도록 선정).
- **다이내믹 타입**: 시스템 글자 크기 존중(고정 px 남용 금지).
- **터치 타깃**: 최소 44×44pt.
- **스크린리더**: VoiceOver/TalkBack 라벨(단계·번호·액션). device_id 등 민감값 노출 금지.
- **reduce motion**: 시스템 설정 시 애니메이션 축소/제거.

## 애니메이션
- fade-in, 짧은 slide만 허용. 글로우·바운스·연속 펄스 금지. reduce motion 존중.

## 아이콘
- 단순 라인 아이콘(strokeWidth 1.5). 둥근 배경 박스로 감싸지 않는다.
