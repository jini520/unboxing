# UI 디자인 가이드

> 대상: Expo (React Native). 웹 Tailwind가 아니라 RN `StyleSheet`/스타일 객체 기준.
> **테마는 시스템 외형(light/dark)을 따르며 라이트가 설계 기준**(→ ADR-016). 색은 하드코딩하지 말고 아래 **시맨틱 토큰**(라이트/다크 값)으로 쓴다.
> 색상·수치는 **초안** — Phase 1 UI 구현 step에서 확정한다. 원칙·안티패턴·접근성·상태/테마 규칙은 확정.
> 기능/플로우/스토어 정책은 `PRD.md`, 데이터/엔드포인트는 `ARCHITECTURE.md`.

## 디자인 원칙
1. **도구처럼 보여야 한다.** 마케팅이 아니라 매일 쓰는 배송 추적 대시보드. 신뢰감 있고 깔끔하게.
2. **시스템 테마를 따르고, 라이트를 기준으로 설계한다.** 대중 택배 앱의 표준 기대치는 라이트. 다크는 동등 품질로 지원(다크 전용/디폴트 아님).
3. **마찰 최소.** 등록은 두세 탭 안에. 불필요한 모달·온보딩 단계 금지.
4. **상태는 색에만 의존하지 않는다.** 색 + 아이콘 + 텍스트 라벨을 항상 함께(접근성).

## AI 슬롭 안티패턴 — 하지 마라
(RN 맥락으로 적응. CSS 용어는 RN 대응으로 읽을 것.)

| 금지 사항 | 이유 |
|-----------|------|
| glass morphism (expo-blur로 카드 배경 흐림 장식) | AI 템플릿의 가장 흔한 징후 |
| 그라데이션 텍스트 | AI가 만든 SaaS 랜딩의 1번 특징 |
| "Powered by AI" 류 배지 | 기능이 아니라 장식. 사용자에게 가치 없음 |
| 글로우 애니메이션 (네온 shadow 펄스) | 네온 글로우 = AI 슬롭 |
| 보라/인디고 브랜드 색상 | "AI = 보라색" 클리셰 |
| 모든 카드에 동일한 큰 borderRadius(2xl 류) | 균일한 둥근 모서리는 템플릿 느낌 |
| 과한 그림자/elevation 남발 | 라이트에서 특히 지저분. 그림자는 아주 옅게만 |
| 배경 그라데이션 orb (blur 원형 장식) | 모든 AI 랜딩 페이지에 있는 장식 |

## 화면 구성

| 화면 | 내용 |
|---|---|
| **택배함**(주화면·하단 탭) | 제목 **"택배함"** + 페이지 설명(마지막 업데이트 문구 아님) + ＋버튼(등록). 마지막 업데이트(신선도)는 **첫 카드 위 오른쪽**. 송장 카드 스택(메모 포함) + 당겨서 새로고침. 멀티선택 모드 헤더 |
| **상세** | **페이지 타이틀 = 메모**(없으면 등록일 default — 일반 페이지 제목처럼 헤더에, 택배사 줄 **위**·사용자 요구) → ①택배사·전체번호(좌)+받는분(우) 한 줄 → ②현재 상태 문구(상단 중앙·크게·친절 교정, '상품' 통일) → ③단계 인디케이터(점 **위에** 단계 아이콘·크게) → ④타임라인(실시간, 좌우 패딩) → 삭제. **메모는 본문에 인라인 박스(textarea) 없음 — 헤더 연필 → 모달에서만 편집**(사용자 요구, 회귀 금지) |
| **등록** | 인앱 헤더(작은 title "운송장 번호 등록" + 자동채움 설명) → 번호 입력 → **(입력창 하단) 클립보드 제안** → 택배사 자동추정/확인 → 등록. 미지원이면 딥링크 카드 |
| **온보딩** | 푸시 권한 사전 안내(priming) → 시스템 요청 |
| **설정**(하단 탭) | 알림 토글 · 테마(시스템/라이트/다크) · 개인정보처리방침(인앱 화면+웹 링크) · **모든 데이터 삭제**(메모 포함) · 앱 버전 |
| **빈 상태** | 가치 제안 한 줄 + 첫 등록 CTA (중앙 정렬 허용) |

- **네비게이션**: 하단 탭 2개(택배함·설정). 상세/등록/온보딩/개인정보처리방침은 탭 위로 push 되는 화면. **네이티브 헤더는 끄고**(headerShown:false) 각 화면이 공용 `ScreenHeader`(배경 없는 아이콘-only 뒤로가기 + 선택적 title/설명)를 그린다 — iOS 26 헤더 버튼 glass 배경 회피.
- **택배함 탭 아이콘**: **닫힌 상자(Package)로 통일** — 활성/비활성 동일 글리프, 색만 토큰으로 구분(열린 상자 금지, 사용자 요구).
- **단계 인디케이터(상세)**: happy-path 5단계(등록→집화→이동중→배송출발→배송완료)를 가로 스텝퍼로. **각 단계 아이콘을 점 위에** 두고(점은 트랙에 유지), 현재=강조 색·크게, 지난=중립, 이후=비활성. off-track(미등록·기타=pre, 예외=경고)은 선형 매핑 금지.

## 테마 & 색상 (시맨틱 토큰 — 라이트 기준)

> 컴포넌트는 hex가 아니라 토큰명을 참조. 라이트가 기본, 다크는 시스템이 다크일 때.

### 배경 / 보더
| 토큰 | 라이트 | 다크 |
|------|------|------|
| `bg/page` | `#ffffff` | `#000000` |
| `bg/surface`(카드) | `#ffffff` | `#1c1c1e` |
| `bg/secondary` | `#f2f2f7` | `#2c2c2e` |
| `border` | `#e5e5ea` | `#38383a` |

- 라이트에선 페이지·카드가 모두 흰색이므로 카드는 **옅은 보더(+아주 약한 그림자)**로 구분. 다크에선 배경색 차이로 구분(그림자 금지).
- **값은 애플 iOS 시스템 컬러**: 배경=systemBackground/secondary·tertiarySystemGroupedBackground, `border`=systemGray5(L)·opaqueSeparator(D). 다크 페이지는 OLED true black(`#000000`). hex 단일 출처는 `app/src/theme/tokens.ts`(각 값 옆 Apple 컬러명 주석).

### 대표(브랜드) 색
| 토큰 | 라이트 | 다크 |
|------|------|------|
| `accent`(브랜드) | `#007aff` | `#0a84ff` |
| `onAccent`(accent 위 전경) | `#ffffff` | `#ffffff` |

- **대표 컬러 = 블루**. **상호작용 요소**에 쓴다(브랜드 강조): 기본 버튼·CTA·**활성 탭 틴트**·**선택 상태**(카드 보더·체크박스)·링크.
- **단계 색과 분리**(회귀 금지) — 과거 선택/링크가 `stage.outForDelivery`(배송출발 파랑)를 accent 대용으로 재사용하던 것을 `accent` 로 디커플링했다. `배송출발` 배지는 계속 `stage.outForDelivery` 사용(의미색). **선택/링크/버튼에 `stage.*` 를 직접 쓰지 말 것**.
- **토스트는 예외** — `text/primary` 배경(흑/백) 유지(브랜드색 아님, 사용자 요구).
- accent 위 텍스트/글리프는 **`onAccent`**(두 스킴 모두 흰색 — blue 위 대비 확보). 채워진 accent 버튼에 `bg/page` 를 전경으로 쓰지 말 것(다크에서 blue 위 대비 부족).

### 텍스트
| 토큰 | 라이트 | 다크 |
|------|------|------|
| `text/primary` | `#000000` | `#ffffff` |
| `text/body` | `#3c3c43` | `#ebebf5` |
| `text/secondary` | `#8e8e93` | `#8e8e93` |
| `text/disabled` | `#c7c7cc` | `#48484a` |

- 애플 label 램프: primary=label, body=secondaryLabel(불투명), secondary=systemGray, disabled=systemGray3. ⚠️ `text/secondary`(`#8e8e93`)는 흰 배경 대비 ≈3.5:1 로 **WCAG AA(4.5) 미달** — iOS 정통값이나, 캡션(상대시간·택배사명 12px)에 쓰이므로 대비가 걱정되면 더 진한 회색으로 조정 가능.

### 단계(시맨틱) 색 — 라이트/다크 모두 대비 확보
| 단계 | 라이트 | 다크 |
|------|------|------|
| `배송완료`(성공) | `#34c759` | `#30d158` |
| `배송출발`(임박) | `#007aff` | `#0a84ff` |
| `이동중`(진행 강조) | `#e0a500` | `#ffd60a` |
| `예외`(에러) | `#ff3b30` | `#ff453a` |
| `등록`·`집화`·`기타`(중립) | `#8e8e93` | `#8e8e93` |
| `미등록`(비활성) | `#aeaeb2` | `#636366` |

- 애플 시맨틱: 완료=systemGreen, 출발=systemBlue, 이동중=systemYellow, 예외=systemRed, 중립=systemGray, 미등록=systemGray2. `배송출발`이 `accent` 와 동색(둘 다 systemBlue)이나 토큰은 분리 유지(디커플링 규칙).
- **이동중 = 노란색 계열**(`stage.inTransit`, 사용자 요구) — 중간단계 중 유일하게 색 부여. ⚠️ 애플 systemYellow(`#ffcc00`)는 **흰 카드 위 대비 ≈1.5:1 로 라벨이 거의 안 보여**, 라이트는 톤 내린 골드(`#e0a500`, ≈2.2:1)로 둔다(다크는 systemYellow 정값 `#ffd60a`). `집화` 는 색을 바꾸지 않고(중립 유지) **아이콘만** 차별화.

## 단계 배지 (색 + 아이콘 + 라벨 — 색 단독 금지)

| 단계 | 아이콘(예) | 라벨 |
|---|---|---|
| `배송완료` | ✓ check(초록) | 배송완료 |
| `배송출발` | truck(파랑) | 배송출발 |
| `이동중` | mappin(노랑) | 이동중 |
| `예외` | alert(빨강) | 예외 |
| `집화` | box/Package(중립) | 집화 |
| `등록`·`기타` | dot(중립) | 각 단계명 |
| `미등록` | clock | 미등록 |

- `집화`·`이동중` 은 중간단계지만 식별성을 위해 고유 아이콘 부여(나머지 중간단계 `등록`·`기타` 는 중립 dot). 카드 배지 소스는 `STAGE_META`(StageBadge.tsx), 상세 스텝퍼는 별도 `STEP_ICONS`(StageProgress.tsx).

색은 위 "단계(시맨틱) 색"을 단일 출처로(ARCHITECTURE 상태 정규화 7단계와 1:1).

## 상태별 UI (로딩/빈/에러 — 누락 금지)

| 상태 | 표시 |
|---|---|
| 로딩(목록) | 스켈레톤 카드(좌측 정렬), 캐시 있으면 캐시 먼저 + 상단 새로고침 인디케이터 |
| 로딩(상세 타임라인) | 단계 배지는 즉시(캐시), 타임라인은 스켈레톤 |
| 빈 목록 | 간결한 가치 제안 + 등록 CTA(중앙 정렬) |
| 네트워크 오프라인 | 상단 **오프라인 배너**(비파괴적), 캐시 표시, 변경 액션 비활성+사유 |
| 조회 실패(상세 upstream) | 마지막 알려진 단계 + "타임라인을 못 불러왔어요, 다시 시도" |
| 미지원 택배사 | "자동 추적 미지원" + 택배사 조회 **딥링크** 버튼 |
| 등록 실패 | 인라인 에러(코드 노출 금지) + 재시도, 입력값 보존 |
| 알림 비활성(권한 거부) | 상단 작은 배너 "알림 꺼짐 — 켜기"(설정 유도) |

## 송장 카드 해부
```
[단계 배지]                                  [음소거 아이콘?] [상태 변경 후 상대시간]
메모(중단 · primary 색, 없으면 "{YYYYMMDD}에 등록한 상품")  ← 좌측정렬(가운데 정렬 X)
택배사명 · 운송장 전체번호                                ← 하단 · 작은 폰트(12)
```
- 좌측 정렬. 단계 배지=색+아이콘+짧은 라벨. 상대시간은 **상태 변경 시각**(`statusChangedAt`) 기준.
- **현재 상태는 StageBadge 로만 표시(회귀 금지)**: 단계 배지(색+글리프+라벨, 예 "이동중")가 곧 상태 표시다. **별도 "현재 상태 메세지" 텍스트 줄(STAGE_SUMMARY·STAGE_STATUS_MESSAGE)을 카드에 추가하지 말 것** — StageBadge 와 역할이 겹쳐 제거하기로 정함(사용자 요구). 위치(허브)는 목록 데이터에 없어 카드 표시 불가(상세 화면만 "이동 중 (허브명)"). 참고: `STAGE_SUMMARY` 상수 자체는 보존(상세 폴백 등 — 임의 제거 금지).
- **메모(중단 · 로컬 전용 · 미전송)** — tracker.delivery 에 상품명이 없어 사용자가 직접 적는 식별 정보. **중단 배치 · 좌측정렬**(가운데 **정렬** 금지 — 사용자가 말한 "가운데"는 **위치**[중단]이지 text-align 이 아니다). 색은 **primary**(default 문구라도 **회색 금지**). 없으면 등록일 기반 default(`{YYYYMMDD}에 등록한 상품` — 예 "20260619에 등록한 상품") — placeholder 가 아니라 **대체 문구**(항상 한 줄). 카드와 **상세 페이지 타이틀**이 같은 문구를 쓰도록 `defaultMemoText()`(memo.ts) 단일 출처. 편집은 상세 헤더 연필 → 모달. 저장은 로컬 AsyncStorage.
- **택배사 · 운송장 전체번호(하단)** — 보조 정보라 **하단 · 작은 폰트(12)**. 운송장 번호는 **전체** 표기(본인 데이터, 끝4자리 X), 길면 잘림 없이 줄바꿈 허용.
- **양방향 스와이프 — 2단계 + 넓은 버튼**: **좌측 스와이프=삭제**, **우측 스와이프=음소거**.
  - 1단계(첫 스와이프): 버튼을 **노출(rest)만** — 길게 스와이프해도 실행되지 않는다. 버튼 **탭**으로도 실행.
  - 2단계: **노출된 상태에서 같은 방향으로 추가로 `SWIPE_AGAIN_PX`(기본 80px)만큼 더 밀면** 버튼 탭과 동일 실행. **민감도는 이 상수 하나로 조절**(클수록 둔감). `openedRef`(onSwipeableOpen/Close)로 "이미 열림"을 게이트해 **첫 스와이프는 실행 안 됨**.
  - **CRITICAL(회귀 금지) — 구현 방식**: 거리 측정은 반드시 **base `AnimatedValue` 인 `dragX`**(제스처 raw translationX, swipeable 내부 `state.dragX`)에 `addListener`. render 콜백이 주는 `progress`(1번째)·`transX`(2번째)는 **interpolation 이라 RN Animated 가 리스너를 발화시키지 않는다**(base Value 만 `__callListeners` 호출). 또 Swipeable 기본 `useNativeAnimations:true`(native driver)면 JS 리스너가 아예 안 오므로 **`useNativeAnimations={false}` 필수**. `onSwipeableWillOpen`-재열림 단독 트리거는 거리 임계가 없어 **과민**(회귀).
  - **CRITICAL(회귀 금지) — 실행 타이밍**: 임계 도달 시 **제스처 진행 중에 액션/`close()`를 호출하면 안 된다** — 진행 중 pan 과 close 애니메이션이 충돌해 **카드가 떨리고**, 손을 떼면 `handleRelease` 가 다시 열어 **1단계에 잔류**한다. dragX 리스너는 임계 도달 시 **예약(pendingFireRef)만** 하고, **정착 시점 `onSwipeableWillOpen` 에서 실행+`close()`**. 구현 `app/src/components/ShipmentCard.tsx`.
  - 참고: 이 `Swipeable`(react-native-gesture-handler legacy)는 **deprecated**(Reanimated 버전 권장)이나 동작엔 문제 없음. 마이그레이션은 `react-native-reanimated` 신규 의존 추가 + 스와이프 재작성이 필요해 **별도 작업**(미설치 상태).
  - **과스와이프 제한(회귀 금지)**: rest(버튼 노출) 너머에서 RNGH `transX` 는 clamp 가 아니라 `1/overshootFriction` 기울기로 **연장**되므로, 기본값 1 이면 카드가 버튼 폭 너머로 끝없이 따라온다(사용자 보고 "너무 많이 스와이프됨"). **`overshootFriction={OVERSHOOT_FRICTION}`(=8)** 로 둔화해 카드를 버튼 폭 근처에 거의 고정한다(라이브러리 권장 "8 or above"). ⚠️ 이 값은 **시각 `transX` 에만** 작용하고 2단계 판정 기준 raw `dragX` 엔 영향 없어 `SWIPE_AGAIN_PX` 는 그대로 동작 — **overshoot 자체를 끄는 방식으로 되돌리지 말 것**(2단계 회귀). 구현 `app/src/components/ShipmentCard.tsx`.
  - **CRITICAL(회귀 금지) — 배경 폭**: 컬러 배경은 노출 가장자리에 앵커한 **한정 폭**(`ACTION_BG_WIDTH`≈120, 음소거=좌측 `actionBgLeft`·삭제=우측 `actionBgRight`)로 깐다. 콘텐츠(아이콘+라벨)는 고정폭(`actionBtn` 96, 스냅 폭). **`StyleSheet.absoluteFill`(카드 폭 전체) 풀폭으로 되돌리지 말 것** — RNGH 패널이 컨테이너 전폭(absoluteFill)이라, 풀폭 배경 + 컨테이너 `overflow:visible`(카드가 거터 넘어 끝까지 밀리게 한 설정)이 만나면 **토글 실행 후 카드 복귀 전환 때 반대편 모서리로 컬러가 샌다**(사용자 보고). `transX` 가 `leftWidth(≈버튼폭)+overshoot(~1px)` 로 클램프돼 노출 폭이 버튼 폭을 안 넘으므로 한정 폭이면 과스와이프 틈도 없고 far edge 가 항상 카드 아래라 안 샌다. **고정폭 배경 자체는 유지**(콘텐츠 96 shrink/clip 금지)·overshoot 유지. 구현: `app/src/components/ShipmentCard.tsx`.
  - **버튼 색/아이콘(회귀 금지)**: **음소거=`accent`(브랜드 블루) 배경**, **삭제=`stage.exception`(빨강) 배경**, 두 버튼 모두 글리프·라벨은 **`onAccent`(흰색)**. 아이콘은 **채움(fill) 글리프**(`BellFill`/`BellOffFill`/`TrashFill` — stroke 라인 버전 아님, 사용자 요구). 카드의 음소거 표시(작은 14px)는 그대로 라인 `BellOff`. **중립 `bg/secondary` 배경 / stroke 아이콘으로 되돌리지 말 것**.
  - 삭제는 Undo 토스트로 복구(확인 다이얼로그 없음), 음소거는 낙관 토글+실패 시 롤백. 탭 → 상세.
- **롱프레스 → 멀티선택**: 체크박스 표시·탭=선택 토글(상세 이동 X)·스와이프 비활성.
  - 헤더 1행: **[취소] · {N}개 선택(가운데) · [일괄삭제]** — 카운트는 화면 중앙(양 끝 동일 폭).
  - 헤더 2행(아래): **전체 선택 / 전체 해제**(전체 해제는 선택만 비우고 선택 모드 유지). 일괄삭제=확인 다이얼로그·**Undo 없음**·부분 실패 복원. 0개 선택이어도 취소/뒤로 전엔 유지.
  - **전체 선택·전체 해제 스타일(회귀 금지)**: **취소 버튼과 동일한 중립 회색**(`text/secondary`)·**emphasis 없음**(굵게/강조색 금지) — 보조 액션이라 눈에 덜 띄게(사용자 요구).
- 음소거 카드는 한 켠에 음소거 아이콘(색 단독 아님, a11y "알림 꺼짐").
- 택배사 식별: 택배사명 + (선택) 택배사 색 점/로고 — 로고 사용 시 **상표 가이드 준수**(임의 변형·왜곡 금지, 미확보 시 텍스트/중립 아이콘으로 대체).

## 디자인 프리미티브 (간격·라운드·타이포 — 단일 출처)

> 스킴(light/dark) **무관** 공통 수치. 색은 위 시맨틱 토큰, 이 값들은 `app/src/theme/layout.ts` 에 모은다. 컴포넌트는 StyleSheet 리터럴 대신 이 토큰을 참조한다(공통 값 단일 출처). 스케일에 없는 1회성 값(원형 점 반지름=width/2·스와이프 버튼 폭·아이콘 `size`·`lineHeight` 등)은 컴포넌트 로컬 리터럴로 둔다.

- **spacing**(padding·margin·gap — 4단위): `xs 4 · sm 8 · md 12 · lg 16 · xl 24 · xxl 32`. 페이지 좌우 거터=`lg`(16), 섹션 간격=`xl`~`xxl`.

- **fontWeight**: `medium 500 · semibold 600 · bold 700`.

## 인터랙션
- **당겨서 새로고침**: 목록·상세. 오프라인이면 안내.
- **양방향 스와이프(버튼 노출 → 탭/추가 스와이프 실행)**: **좌측 스와이프=삭제**(reveal + Undo 토스트로 실수 복구, 서버 반영은 Undo 만료 후 — 별도 확인 다이얼로그 없음), **우측 스와이프=음소거**(낙관 토글 + 실패 시 롤백). 첫 스와이프는 버튼만 노출(정지).
- **롱프레스 멀티선택 + 일괄삭제**: 롱프레스로 선택 모드 진입(체크박스·전체선택·취소). 일괄삭제는 **확인 다이얼로그 + Undo 없음**(부분 실패 시 해당 항목 복원·안내). 단건 스와이프 삭제만 Undo 유지.
- **클립보드 제안**: 등록 화면 진입 등 명시 시점에만, 제안만(자동 등록 금지 — iOS 배너 정책, PRD 권한). 위치는 **운송장 번호 입력창 하단**(상단 아님 — 사용자 요구).
- **토스트/에러**: 일시 오류는 비차단 토스트, 영구 오류는 인라인.
